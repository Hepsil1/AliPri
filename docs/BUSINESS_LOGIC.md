# ⚡ Бизнес-логика — СчётДетей

## 1. Жизненный цикл абонемента

```
┌──────────┐    покупка    ┌──────────┐   used >= total   ┌──────────┐
│  (нет)   │──────────────▶│  ACTIVE   │─────────────────▶│ EXPIRED  │
└──────────┘               └──────────┘                   └──────────┘
                                │  ▲
                      заморозка │  │ разморозка
                                ▼  │
                           ┌──────────┐
                           │  FROZEN   │
                           └──────────┘
```

### Правила:
1. Спортсмен может иметь **только один активный абонемент** одновременно
2. Новый абонемент можно создать только если предыдущий `expired` или не существует
3. При заморозке запоминается `frozen_at` и `freeze_reason`
4. При разморозке `frozen_at` обнуляется, статус → `active`

## 2. Алгоритм отметки посещения

```
Тренер нажимает отметку на спортсмене
          │
          ▼
    Есть ли активный абонемент?
       /          \
     ДА           НЕТ
      │             │
      ▼             ▼
  Абонемент       Тип оплаты?
  заморожен?     /         \
   /     \    single      нет абонемента
  ДА     НЕТ    │              │
  │       │     ▼              ▼
  │       │  Записать как   Показать
  │       │  single_pay     предупреждение:
  │       │  (ввести сумму) "Нет абонемента!"
  │       │
  │       ▼
  │    Показать варианты:
  │    ⭐ Был (present)
  │    🔴 Не был, списать (absent_counted)
  │    ❄️ Не был, заморозка (absent_frozen)
  │
  ▼
  Автоматически:
  статус = absent_frozen
  (занятие НЕ списывается)
```

## 3. Авто-уведомления

### Триггеры:
| Остаток | Действие | Цвет в UI |
|---------|----------|-----------|
| 3+ занятий | Ничего | 🟢 Зелёный |
| 2 занятия | Предупреждение в UI | 🟡 Жёлтый |
| 1 занятие | Жёлтое уведомление | 🟠 Оранжевый |
| 0 занятий | Красное уведомление + текст для Telegram | 🔴 Красный |

### Текст уведомления (шаблон):
```
Привет, [Имя]! 👋
Твой абонемент ([кол-во] занятий) закончился.
Сегодня было крайнее занятие.
Для продления напиши мне!
```

## 4. Журнал тренировки (основной экран)

### Сценарий использования:
1. Тренер открывает приложение на телефоне
2. Выбирает группу (или автоматически по расписанию)
3. Видит список спортсменов группы
4. Нажимает на спортсмена → выбирает статус
5. Рядом с именем видит остаток абонемента (цветной бейджик)
6. После отметки всех → видит сводку

### Визуализация остатка абонемента:
```
[████████] 8/8  — полный (зелёный)
[██████░░] 6/8  — норма (зелёный)  
[████░░░░] 4/8  — половина (зелёный)
[██░░░░░░] 2/8  — мало! (жёлтый)
[█░░░░░░░] 1/8  — последнее! (оранжевый)
[░░░░░░░░] 0/8  — истёк (красный)
```

---

## 5. Разовые оплаты

- Спортсмены без абонемента отмечаются как `single_pay`
- При отметке вводится сумма оплаты
- В отчётах видна общая сумма разовых оплат за период

---

## 6. Спортсмен может переходить между группами

- При переносе в другую группу абонемент **сохраняется**
- История посещений **сохраняется**
- Меняется только `group_id`

---

## 7. Ограничения и защита данных

- Нельзя удалить группу, если в ней есть спортсмены
- Нельзя удалить спортсмена с активным абонементом (только деактивировать)
- Все действия пишутся в `attendance` — полная аудиторская история
- SQLite WAL mode для безопасности записи
